## 四、字体风格匹配方案（v23.0 模块化）

> **v23.0**：字体资源已模块化拆分至 `references/fonts/`。按 `fonts/font-match-engine.md` 规则，根据风格编号自动匹配分类ID，加载对应模块文件获取 CDN + CSS。PC 端字体推荐由各模块文件自维护。

### 4.1 风格编号→分类ID速查

```
#1→F1  #2→F1  #3→F1  #4→F1  #5→F1  #6→F1
#7→F2  #8→F6  #9→F2  #10→F2 #11→F3 #12→F2
#13→F3 #14→F1 #15→F1+F8 #16→F7 #17→F2 #18→F1+F8
#19→F1 #20→F5 #21→F4 #22→F2 #23→F4 #24→F1
#25→F7 #26→F3 #27→F1 #28→F1 #29→F1 #30→F2 #31→F5
```

### 4.2 PC 端字体推荐（精简表）

> 详细参数见各 `fonts/` 模块文件末尾的 `window.RECOMMENDED_*_FONT` 声明。

| 分类ID | 推荐标题字体（PC） | 推荐内容字体（PC） |
|:------:|-------------------|-------------------|
| F1 | 思源宋体 | 微软雅黑 |
| F2 | 思源黑体 | 微软雅黑 |
| F3 | 思源黑体 | 微软雅黑 |
| F4 | 微软雅黑 | 微软雅黑 |
| F5 | 楷体 | 楷体 |
| F6 | 楷体 | 楷体 |
| F7 | 思源黑体 | 微软雅黑 |
| F8 | 思源宋体 | 微软雅黑 |

> **原则**：经典/文学/人文用思源宋体（衬线），现代/科技/极简用思源黑体（无衬线），书法/水墨/手写用楷体，圆润/可爱用微软雅黑。

## 五、CSS 颜色变量速查（示例）

> **v14.0 起**：31 种风格的完整 CSS 变量已预计算在 `assets/styles-css/{01-31}.css` 中，AI 注入时直接读取对应 CSS 文件即可。以下为部分风格参考示例：

```css
/* #1 史诗金箔 */

--bg: linear-gradient(135deg, #F5F0E8, #EDE0C8, #D9C7A0);
--accent: #8B6914;
--text: #2C2216;
```
