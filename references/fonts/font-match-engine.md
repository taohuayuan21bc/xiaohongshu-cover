# 字体智能匹配引擎（v23.0）

> **用途**：根据内容品类→调性→风格编号，自动匹配最佳字体分类。首轮方案预览时使用（不加载字体模块），选定方案后按需加载。

---

## 一、内容→字体自动匹配规则

### 1.1 品类→字体倾向

| 品类 | 字体倾向 | 分类ID | 原因 |
|------|---------|:------:|------|
| 文学书评 | 衬线经典 | F1 | 文学气质，衬线字体传递质感 |
| 古典文学/国学 | 楷体传统 | F6 | 书法感，东方韵味 |
| 知识干货/职场 | 无衬线现代 | F2 | 信息清晰，阅读效率优先 |
| 科技/AI/数码 | 未来科技 | F7 | 科技感，几何字体加强调性 |
| 美妆/穿搭/超扁平 | 圆润可爱 | F4 | 年轻化，亲和力 |
| 情感/治愈 | 圆润可爱 或 手写文艺 | F4/F5 | 温暖感，手写传达温度 |
| 青年/自媒体/爆款 | 极粗醒目 | F3 | 冲击力，粗体传递力量 |
| 巴洛克/维多利亚 | 哥特古典 | F8 | 华丽装饰感，古典气质 |
| 赛博朋克 | 未来科技 | F7 | 霓虹+科技感字体 |

### 1.2 调性→字体倾向

| 调性 | 字体倾向 | 分类ID |
|------|---------|:------:|
| 严肃专业 | 无衬线现代 | F2 |
| 活泼有趣 | 圆润可爱 或 极粗醒目 | F4/F3 |
| 治愈温暖 | 手写文艺 或 衬线经典 | F5/F1 |
| 高级质感 | 衬线经典 或 无衬线现代 | F1/F2 |
| 知识硬核 | 无衬线现代 | F2 |
| 国风雅致 | 楷体传统 | F6 |

### 1.3 字数→字体影响

| 标题字数 | 字体倾向 | 原因 |
|---------|---------|------|
| ≤6 字 | 极粗醒目(F3)/衬线经典(F1) | 大字可用装饰性强字体 |
| 7-12 字 | 全类别通用 | 标准适配 |
| >12 字 | 无衬线现代(F2) 优先 | 长文字阅读效率优先 |

## 二、风格编号→字体分类速查

```
#1 → F1  #2 → F1  #3 → F1  #4 → F1  #5 → F1  #6 → F1
#7 → F2  #8 → F6  #9 → F2  #10→ F2  #11→ F3  #12→ F2
#13→ F3  #14→ F1  #15→ F1+F8  #16→ F7  #17→ F2  #18→ F1+F8
#19→ F1  #20→ F5  #21→ F4  #22→ F2  #23→ F4  #24→ F1
#25→ F7  #26→ F3  #27→ F1  #28→ F1  #29→ F1  #30→ F2  #31→ F5
```

> **注**：#15/#18 同时加载 F1（衬线）+ F8（哥特装饰），共两个模块。

---

## 三、加载工作流

```
Step A：内容分析 → 确定风格编号
Step B：查§二速查表 → 确定字体分类ID
Step C：查 font-catalog.md §一 → 确定分类文件名
Step D：加载对应 .md 文件 → 获取 CDN link + CSS 注入代码
Step E：INJECT_STYLE_CSS 时注入字体内容
```

**Token 消耗**：
- 本文件始终在上下文：~250 tokens
- 加载 1 个字体模块：~80 tokens（v25.0：含 preconnect + display=swap）
- 合计：~330 tokens（vs 旧版 ~3000 tokens，节省 ~89%）
